# Plan: AutoAim 重构 — 从 MegaTag2 姿态解算 → 原始 AprilTag 相机空间控制

## 目标

把自瞄从"场地坐标系姿态解算"改为"原始AprilTag 相机空间直接控制"：
- 读取 limelight-back 上的 `rawfiducials`，过滤 ID {2, 9, 10, 11}
- 选最近的 tag，用 txnc PID 闭环锁云台，用 tync 查表定 pivot 角度
- 飞轮不接入，保持手动

---

### 2. 修改文件

#### `src/main/java/frc/robot/commands/AutoAimCommand.java` — **重写**
新逻辑：
1. `LimelightHelpers.getRawFiducials("limelight-back")` 获取所有检测到的 AprilTag
2. 过滤 ID ∈ {2, 9, 10, 11}，选 `distToRobot` 最小的那个
3. **Turret（左右）**：PID 闭环把 txnc 归零
   - 读当前云台角度 `turret.getCurrentAngle()`
   - 计算目标角度 `target = currentAngle + kP * txnc`
   - `m_turret.setTargetAngle(target)`
4. **Pivot（上下）**：tync → 距离 → 查表
   - 水平距离 `dist = 0.62 / Math.tan(Math.toRadians(-tync))` （ty 通常为负值，tag 在视线上方）
   - 用 `ShooterLookupConstants` 的距离查表插值得到 pivot 角度
   - `m_pivot.setTargetAngle(lookupAngle)`
5. **未找到 tag 时**：保持当前位置不动（不做任何 setTargetAngle）

## 查表设计

`ShooterLookupConstants` 改为按 **水平距离(m)** 查表：

```java
public static final class ShooterLookupConstants {
    // 水平距离 (m) → Pivot 角度 (degrees)
    public static final double[] DISTANCE_TO_PIVOT_DISTANCE = {1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0};
    public static final double[] DISTANCE_TO_PIVOT_ANGLE =   {2.0, 5.0, 10.0, 15.0, 20.0, 25.0, 30.0};
    // 这些值为占位，需要实测调参
}
```

查表插值方法（写在 AutoAimCommand 或一个工具类中）：
- 距离小于最小值 → 返回最小角度
- 距离大于最大值 → 返回最大角度
- 中间值 → 线性插值

## 验证方案

1. `.\gradlew build` 编译通过
2. 部署到机器人后：
   - 按住 rightTrigger → 观察 SmartDashboard 上的 turret 角度和 txnc 值
   - 云台应自动转向使 txnc 趋近 0
   - pivot 角度应随距离变化（走近/走远观察）
   - 松手后 turret 和 pivot 保持当前位置
3. 调参重点：
   - `TURRET_KP`：太大会振荡，太小会跟踪慢
   - 查表值：实测各距离下的最佳 pivot 角度填入
