## T1b: Create LimelightVisionSubsystem

**Goal:** New subsystem that reads both Limelight 3Gs via NetworkTables and exposes odometry + targeting data.

**File to create:** `src/main/java/frc/robot/subsystems/LimelightVisionSubsystem.java`

**Key design:**
```
class LimelightVisionSubsystem extends SubsystemBase

Constructor:
  - Get NetworkTableInstance, get table references for BACK_LIMELIGHT and FRONT_LIMELIGHT
  - Initialize pose storage (estimated robot pose from back limelight)

periodic():
  - BACK LIMELIGHT (odometry):
    - Read "botpose_orb" (Megatag 2 output: [x, y, z, roll, pitch, yaw, latency, tagCount, ...])
    - If tagCount > 0, compute pose from x, y, yaw
    - Apply standard deviations based on tag count (use VisionConstants.SINGLE/MULTI_TAG_STD_DEV)
    - Call swerveDrive.addVisionMeasurement(pose, timestamp)
  - FRONT LIMELIGHT (targeting):
    - Read "tx" (horizontal offset to target, degrees)
    - Read "ty" (vertical offset to target, degrees)
    - Read "tv" (valid target exists: 0 or 1)

Getters:
  - boolean hasTarget() — tv == 1
  - double getTargetTx() — horizontal offset (degrees), 0 = centered
  - double getTargetTy() — vertical offset (degrees)
  - double getEstimatedDistance() — calculate from ty using trig:
      distance = (FRONT_CAM_HEIGHT - TARGET_HEIGHT) / tan(toRadians(FRONT_CAM_MOUNT_ANGLE + ty))
  - Optional<Pose2d> getEstimatedPose() — from back limelight (for external use)
```

**Pattern reference:** This follows the same `SubsystemBase` pattern as the existing subsystems.

**Key imports:**
```java
import edu.wpi.first.networktables.NetworkTable;
import edu.wpi.first.networktables.NetworkTableInstance;
import edu.wpi.first.networktables.DoubleArrayEntry;
import edu.wpi.first.math.geometry.Pose2d;
import edu.wpi.first.math.geometry.Rotation2d;
```

**Constructor dependency:** Takes `SwerveDrive swerveDrive` reference to call `addVisionMeasurement()`.

**Verification:** Deploy, open Shuffleboard/AdvantageKit, verify `tx` and `tv` values update when a target is visible. Verify `Estimated Distance` reads reasonable values.

## T3: Build Interpolation Framework

**Goal:** Create a lookup utility that converts distance to pivot angle and flywheel RPM.

**File to create:** `src/main/java/frc/robot/util/ShooterLookup.java`

**Pattern:**
```java
package frc.robot.util;

import edu.wpi.first.math.interpolation.InterpolatingDoubleTreeMap;
import frc.robot.Constants.ShooterLookupConstants;

public class ShooterLookup {
    private static final InterpolatingDoubleTreeMap distanceToPivot = new InterpolatingDoubleTreeMap();
    private static final InterpolatingDoubleTreeMap distanceToRPM = new InterpolatingDoubleTreeMap();

    static {
        // Load from Constants (placeholder values — tune on real robot)
        for (int i = 0; i < ShooterLookupConstants.DISTANCE_TO_PIVOT_DISTANCE.length; i++) {
            distanceToPivot.put(
                ShooterLookupConstants.DISTANCE_TO_PIVOT_DISTANCE[i],
                ShooterLookupConstants.DISTANCE_TO_PIVOT_ANGLE[i]
            );
        }
        for (int i = 0; i < ShooterLookupConstants.DISTANCE_TO_RPM_DISTANCE.length; i++) {
            distanceToRPM.put(
                ShooterLookupConstants.DISTANCE_TO_RPM_DISTANCE[i],
                ShooterLookupConstants.DISTANCE_TO_RPM_RPM[i]
            );
        }
    }

    /** Returns interpolated pivot angle (degrees) for a given distance (meters). */
    public static double getPivotAngle(double distanceMeters) {
        return distanceToPivot.get(distanceMeters);
    }

    /** Returns interpolated flywheel RPM for a given distance (meters). */
    public static double getRPM(double distanceMeters) {
        return distanceToRPM.get(distanceMeters);
    }
}
```

**Verification:** Unit test or manual test — call `ShooterLookup.getPivotAngle(2.5)` and confirm it returns a value between the 2m and 3m entries.

## T4: Create AimAndSpinUpCommand

**Goal:** The core "aim and shoot" command that ties Vision, Turret, Pivot, and Flywheel together.

**File to create:** `src/main/java/frc/robot/commands/AimAndSpinUpCommand.java`

**Pattern:**
```java
package frc.robot.commands;

import edu.wpi.first.wpilibj2.command.Command;
import frc.robot.subsystems.FlywheelSubsystem;
import frc.robot.subsystems.LimelightVisionSubsystem;
import frc.robot.subsystems.PivotSubsystem;
import frc.robot.subsystems.TurretSubsystem;
import frc.robot.util.ShooterLookup;

public class AimAndSpinUpCommand extends Command {

    private final TurretSubsystem turret;
    private final FlywheelSubsystem flywheel;
    private final PivotSubsystem pivot;
    private final LimelightVisionSubsystem vision;

    public AimAndSpinUpCommand(TurretSubsystem turret, FlywheelSubsystem flywheel,
                                PivotSubsystem pivot, LimelightVisionSubsystem vision) {
        this.turret = turret;
        this.flywheel = flywheel;
        this.pivot = pivot;
        this.vision = vision;
        addRequirements(turret, flywheel, pivot); // Claim subsystems
    }

    @Override
    public void execute() {
        // 1. AIM TURRET: Drive turret to center the target (tx -> 0)
        if (vision.hasTarget()) {
            // Simple P-loop: turret offset = -tx (negative because turret must rotate opposite)
            turret.setTargetAngle(-vision.getTargetTx());
        }

        // 2. LOOKUP SHOOTER PARAMS from distance
        double distance = vision.getEstimatedDistance();
        double pivotAngle = ShooterLookup.getPivotAngle(distance);
        double rpm = ShooterLookup.getRPM(distance);

        // 3. COMMAND PIVOT and FLYWHEEL
        pivot.setTargetAngle(pivotAngle);
        flywheel.setTargetRPM(rpm);
    }

    @Override
    public void end(boolean interrupted) {
        // STOW everything when trigger released
        turret.setTargetAngle(0);     // Return turret to center
        pivot.setTargetAngle(0);      // Stow pivot
        flywheel.stop();              // Stop flywheels
    }

    @Override
    public boolean isFinished() {
        return false; // Only ends when interrupted (trigger released)
    }
}
```

**Key design decisions:**
- Turret aiming uses a simple proportional approach: `setTargetAngle(-tx)`. The TurretSubsystem's MotionMagic handles smooth profiling. This is the simplest approach — the turret continuously tracks the target. If a PID-on-error approach is needed later, the `TurretConstants` PID can be tuned.
- Distance calculation uses `LimelightVisionSubsystem.getEstimatedDistance()` from ty.
- `addRequirements()` ensures subsystem mutual exclusion.

**Verification:** Hold Right Trigger with a target visible. Turret should track, pivot should move to looked-up angle, flywheels should spin at looked-up RPM. Release trigger — everything stows.

### T5b: Drive-By-Aiming Mode

When aiming, the driver can still translate with the left stick, but the **chassis rotation** is overridden by the Limelight's front camera tx offset. This allows the driver to strafe toward the target while the turret + chassis auto-aim.

**Add to SwerveSubsystem:**
```java
/**
 * Drive with manual translation but vision-controlled rotation.
 * Used during aiming: left stick translates, Limelight tx controls chassis rotation.
 */
public Command driveFieldOrientedAiming(Supplier<ChassisSpeeds> translationSupplier,
                                         LimelightVisionSubsystem vision) {
    return run(() -> {
        ChassisSpeeds speeds = translationSupplier.get();
        if (vision.hasTarget()) {
            // Override rotation with vision P-loop
            speeds = new ChassisSpeeds(
                speeds.vxMetersPerSecond,
                speeds.vyMetersPerSecond,
                -vision.getTargetTx() * Constants.OperatorConstants.AIM_P_GAIN
            );
        }
        swerveDrive.driveFieldOriented(speeds);
    });
}
```

**In RobotContainer**, change the aim command to also override the default drive command when active:
```java
// When aiming, replace default drive command with aiming drive
driverXbox.rightTrigger().whileTrue(
    new AimAndSpinUpCommand(turret, flywheel, pivot, vision)
);
// Alternative: use driveFieldOrientedAiming as the default drive stream
// and toggle based on aiming state
```

**Design note:** The cleanest approach is to modify the SwerveInputStream or use a trigger-based toggle. When Right Trigger is held:
1. Default drive command is overridden by `driveFieldOrientedAiming`
2. Turret + Pivot + Flywheel aim and spin up via AimAndSpinUpCommand

**Verification:** Hold Right Trigger. Left stick still drives translation. Chassis and turret auto-aim toward target. Release — normal driving resumes.
